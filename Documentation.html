Part I: The Philosophy

"What were the rules I used to write this for authors?"

1. The Minimum Needed to Write It was a massive task to boil down everything writing software allows you to do into just what an author needs to do. I pared the fat down to the bone. If it doesn't put words on the page, it’s gone.

2. The Cuts (Distractions)

Multiple Fonts: There isn't a need to use multiple fonts in a manuscript. If you need to indicate a computer speaking, use Bold or Italics. Fonts are for professional layout, not for drafting.

Checkboxes & Lists: This isn't a To-Do list application. If you need a checklist, use a notebook.

Colors: Nope. No text colors to play with. No color-coding the tree.

The Corkboard: Sigh. I love that feature in other apps. But it’s a distraction, and it is far too easy to spend hours shuffling index cards when you should be writing scenes.

Version Control: If you are writing a draft, you know that. If you are writing a correction, you know that too. The software doesn't manage "snapshots" because that’s just another form of procrastination.

3. The Essentials (The Tools)

Formatting: Bold, Italic, Underline, and Alignment (Left/Right/Center/Justify) are in there. That’s it.

Structure: A robust Tree structure to keep track of your chapters and scenes? In there.

Focus: Full-screen, distraction-free editing? In there.

Environment: Dark/Light/System palette configuration? In there. For those of us with difficulty seeing, you can pick the font and character size for the whole novel. You need to be comfortable to write.

Safety: Background saving is standard. You don't have to stop typing to save; it happens silently so you never lose your work if the power goes out.

4. The "Fancy" Features (Allowed only because they help flow)

Text-to-Speech: It will read to you if you need it while editing. This is my only nod to "fancy" features because I use it religiously to catch rhythm errors.

Spell Check: Available, but only when you ask for it. No distracting red squiggles while you are trying to be creative.

Typing Sounds: Optional (for those who need the clack-clack of a typewriter to feel productive).

5. The Keyboard Rule Everything the program does is aimed at one thing: Flow. Everything you can do with the mouse is available at the keyboard so you can keep your eyes on the screen and your mind on the story. No bells, bings, or flashes unless you want them (and the flashes are a hard "No," sorry).

6. The Hidden Power (Scripting) The program is simple on the outside, but it hides powerful customization. There is a scripting language built in. It’s not Python or Lua; it’s a language written by me to be intuitive, stack-based, and simple.

The Parity Principle: If you can do it with the mouse or keyboard, a script can do it, too.

The Restriction: You cannot edit scripts inside this program. That’s coding, not writing. If you need a tool, write the script in a text editor and import it. This keeps the "Novel Writer" separate from the "Tool Builder."

When you first start TextSmith, it's ready for you to write. You have a blank slate. The first time you save the novel, TextSmith will ask you what to name it and where to put it (it defaults to your Documents directory).

On the left, you will see an item named <unnamed_0>. This is the start of your novel. You can either write your whole book right here, or create new items to hold your writing "underneath" it (or inside it, if you prefer).

To create an item under (or inside) another item, just hit Alt+N. You'll see a dialog asking you to name the item and create tags for it. Give it a name (like 'Prologue') and hit Enter. We will talk more about tags later; right now, you need to get writing.

You can hit Alt+N as many times as you need to lay out the structure of your novel.

Moving Around:

You can click on any item in the tree to switch to it.

The Keyboard Way: If you are writing and don't want to reach for the mouse, hit Shift+Tab to jump over to the tree, and use your arrow keys to navigate to the item you want. Hit Tab to jump back into the editor and start typing.

TextSmith remembers exactly where you left off in each item.

Saving: Hit Ctrl+S to save the novel, give it a name, and get back to work. TextSmith also autosaves as you go—saving quietly in the background at the intervals you tell it to—so you don't need to worry about losing your work.

There are five menus, (FIle, Edit, Novel, View, and Help). Let's go through them in order and describe what each one does. TextSmith goes to great lengths to only enable menu selections that make sense at any given time. This reduces the what-just-happened p[roblem.

The File Menu

The File menu is where all things file related go.

New - Start a new novel. If the current novel is changed, you'll be given a chance to save it, just get rid of all unsaved changes, or stop the command. The default action is Save, of course!

Open - Open an already existing novel. Like New, it will ask you to save any unsaved changes before you can unknowwing lose your work. The document will be opened and you will be in the exact spot you were at when you last saved it.

Save - Saves your work to the last novel you saved it as. If you've never saved it before, it will ask you for a filename. You can cancel that and it won't be saved, but TextSmith remembers you didn't save it.

Save As - Saves the novel under a new name. You can cancel it at the filename selection point, but again TextSmith remembers you changed the document.

Export - Export the current novel in one of the following formats: EPub,HTML, Markdown, PDF, or RTF (most editors like Microsoft Word and Libre Office Writer can read RTF files).

Page Setup - Set up things like margins, header and footers for printing your documents.

Exit - Leave the program. If there are unsaved, changes, of course the program will keep you from losing any work unless you really want to.

The Edit Menu

The Edit is where you find the typical, and sometimes not so typical, text editing things go.

Undo - Undo the last thing you did. The program keeps track of what you did in somethign like a stack of plates fashion and lets you undo them in the reverse order that you did them.

Redo - If you undid something you wish you hadn't (maybe by overzealous Ctrl+Z keypresses, redo lets you walk that back, putting things back on the pile of plates, so long as you haven't put anything back on the stack yet.

Cut - Take what's been highlighted, copy it to the paste buffer and delete fro the editor, just like about every other program on your computer.

Copy - Same as Cut except the highlighted thing isn't deleted, again like most software you've used before.

Paste - This takes whatever you've put in the paste buffer, from anywhere on you computer, and plops it into TextSmith in the best way it knows how. Which may invovle stripping a lot of fluff out, if it even can be pasted into a text editor. TextSMith does have limited image handling, so give it a try, it might work ...
