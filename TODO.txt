Typewriter sounds:

🎹 The core idea: preload + polyphony + fire‑and‑forget
To get instant, typewriter‑like feedback, you need three things:
1. Preload all sounds into memory
Never load from disk on keypress.
Never recreate a sound object on keypress.
2. Use multiple sound channels (polyphony)
A single  cannot play overlapping sounds.
If you hit keys fast, the sound will cut off.
Solution: a pool of sound players.
3. Trigger sounds on the keypress event, not textChanged
 fires after layout, undo stack, formatting, etc.
That’s too late.
Use .

🔥 The correct Qt class: QMediaPlayer + QAudioOutput (NOT QSoundEffect)
 is simple but has two fatal flaws for your use case:
• 	It has a built‑in ~20–40ms latency on some platforms
• 	It cannot play the same sound twice simultaneously
 with a preloaded buffer is far more responsive and supports polyphony.

🧠 Architecture that will never lag
1. Create a SoundPool class
• 	Preload each sound into memory
• 	Create N players per sound (e.g., 4–8)
• 	Round‑robin through them on each trigger
• 	No allocations, no disk I/O, no blocking
2. Hook into your full‑screen editor’s keyPressEvent
• 	Detect normal keys → “whack”
• 	Detect space → “thunk”
• 	Detect return → “roll + clunk”
• 	Detect margin wrap → “ding + roll + clunk”
Margin wrap detection is easy because you already have your custom layout engine.

🎯 Timing the “ding” on margin wrap
You want the classic typewriter behavior:
• 	As the carriage reaches the right margin → ding
• 	On return → roll + clunk

🧩 Handling fast typists (the real challenge)
A fast typist might generate:
• 	10–15 keypresses per second
• 	overlapping sounds
• 	repeated sounds of the same type
Your sound pool solves this:
• 	Each sound has multiple players
• 	If one is busy, the next one fires instantly
• 	No waiting, no cutting off
This is exactly how games handle gunshots, footsteps, etc.

🎧 Optional: add a tiny random pitch variation
This is how you avoid “machine‑gun sameness”:
• 	±3% pitch variation per keypress
• 	Makes it feel organic
• 	Writers love it
Qt doesn’t expose pitch directly, but you can cheat by:
• 	Pre‑rendering 3–5 variants of each sound
• 	Randomly selecting one from the pool

[SoundPool.h]
#pragma once
#include <QObject>
#include <QMediaPlayer>
#include <QAudioOutput>
#include <QVector>
#include <QUrl>
#include <memory>

class SoundPool : public QObject
{
    Q_OBJECT
public:
    explicit SoundPool(QObject* parent = nullptr);

    enum class Sound {
        KeyWhack,
        SpaceThunk,
        ReturnRoll,
        ReturnClunk,
        MarginDing
    };

    // Preload all sounds
    void load(Sound sound, const QUrl& url, int voices = 4);

    // Play instantly (no lag, no waiting)
    void play(Sound sound);

private:
    struct Voice {
        std::unique_ptr<QMediaPlayer> player;
        std::unique_ptr<QAudioOutput> output;
    };

    struct Bank {
        QVector<Voice> voices;
        int index = 0; // round-robin
    };

    QHash<Sound, Bank> m_banks;
};

[SoundPool.cpp]
#include "SoundPool.h"

SoundPool::SoundPool(QObject* parent)
    : QObject(parent)
{
}

void SoundPool::load(Sound sound, const QUrl& url, int voices)
{
    Bank bank;

    for (int i = 0; i < voices; ++i) {
        Voice v;
        v.output = std::make_unique<QAudioOutput>();
        v.player = std::make_unique<QMediaPlayer>();

        v.player->setAudioOutput(v.output.get());
        v.player->setSource(url);

        // Pre-buffer the audio
        v.player->setLoops(1);
        v.output->setVolume(1.0);

        bank.voices.push_back(std::move(v));
    }

    m_banks.insert(sound, std::move(bank));
}

void SoundPool::play(Sound sound)
{
    if (!m_banks.contains(sound))
        return;

    Bank& bank = m_banks[sound];

    Voice& v = bank.voices[bank.index];
    bank.index = (bank.index + 1) % bank.voices.size();

    // Restart from beginning
    v.player->stop();
    v.player->setPosition(0);
    v.player->play();
}

[on FullScreenEditor keypress]
void FullScreenEditor::keyPressEvent(QKeyEvent* e)
{
    switch (e->key()) {
    case Qt::Key_Space:
        soundPool->play(SoundPool::Sound::SpaceThunk);
        break;

    case Qt::Key_Return:
    case Qt::Key_Enter:
        soundPool->play(SoundPool::Sound::ReturnRoll);

        QTimer::singleShot(80, this, [this]() {
            soundPool->play(SoundPool::Sound::ReturnClunk);
        });
        break;

    default:
        if (!e->text().isEmpty() && !e->text().at(0).isSpace())
            soundPool->play(SoundPool::Sound::KeyWhack);
        break;
    }

    QTextEdit::keyPressEvent(e);

    // Margin wrap detection
    if (cursorRect().right() >= wrapMarginPx)
        soundPool->play(SoundPool::Sound::MarginDing);
}


Scripting ideas:

📄 Export & Formatting
- Scene Outline Export: generate a list of scene titles with summaries/tags.
- POV/Tag Filter Export: export only scenes tagged with a specified keyword.
- Timeline Export: collect scene dates/times from tags and output a chronological timeline.

🔍 Analysis & Consistency
- Word Count per Scene/Chapter: tally words and report totals for pacing analysis.
- Tag Frequency Report: count how often specific tags (e.g. POV, location) appear.
- Dialogue Ratio Check: calculate percentage of dialogue vs. narration in each scene.
- Character Mention Tracker: scan text for character names and produce a per‑scene presence map.

✍️ Text Manipulation
- Smart Find/Replace: replace words/phrases across all scenes with undo‑safe logging.
- Tag Injection: automatically add tags based on regex matches (e.g. detect “Letter:” and tag scene as correspondence).

🛠️ UI & Workflow Automation
- Menu Macro Runner: bind a script to a menu item that runs a sequence of editor commands.
- Batch Scene Reorder: reorder scenes based on tag criteria (e.g. timeline) instead of manual drag.
- Quick Outline Generator: insert a new document section with chapter/scene headings only.
- Delete Empty Scenes: scan tree and remove nodes with no text content.

🎁 Showcase / Fun
- Random Prompt Inserter: insert a writing prompt at the cursor from a supplied list.
- Mood Tagger: scan for sentiment words and tag scenes with “tone: dark/light/romantic.”
- Chapter Title Styler: auto‑apply formatting (caps, underline) to chapter headings.
