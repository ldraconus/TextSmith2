[Display tour - menus]

The Menus
There are five main menus (File, Edit, Novel, View, and Help). Let's go through them in order and describe what each one does.

A quick note: TextSmith goes to great lengths to only enable menu selections that make sense at any given time. This reduces the classic "what-just-happened?" problem by preventing you from clicking things you shouldn't.

The File Menu
The File menu is where all things file-related go.

New: Start a new novel. If your current novel has unsaved changes, TextSmith will ask if you want to save them, discard them, or cancel the command. (The default action is Save, of course!)

Open: Open an existing novel. Just like 'New', it will ask you to save any current changes so you don't unknowingly lose your work. The document will open exactly where you left off when you last saved it.

Save: Saves your work to the current file. If you've never saved it before, it will ask you for a filename. You can cancel if you change your mind, but TextSmith will remember that the document is still unsaved.

Save As: Saves the novel under a brand new name.

Export: Export the current novel into one of the following formats: EPub, HTML, Markdown, PDF, or RTF (most word processors like Microsoft Word and LibreOffice Writer can read RTF files).

Page Setup: Set up your margins, headers, and footers for printing your documents.

Exit: Leave the program. If you have unsaved changes, TextSmith will step in to keep you from losing any work before it closes.

The Edit Menu
The Edit menu is where you find typical—and sometimes not-so-typical—text editing tools.

Undo: Undo the last thing you did. The program keeps track of your actions like a stack of plates, letting you undo them in the exact reverse order you did them.

Redo: If you undid something you wish you hadn't (perhaps from an overzealous string of Ctrl+Z keypresses), Redo lets you walk that back, putting things back on the "stack of plates"—as long as you haven't typed anything new yet.

Cut: Take the highlighted text, copy it to your paste buffer, and delete it from the editor.

Copy: Same as Cut, except the highlighted text is left exactly where it is.

Paste: Takes whatever is in your paste buffer (from anywhere on your computer) and plops it into TextSmith. This may involve stripping out a lot of formatting fluff if it's coming from a complex webpage or program. TextSmith does have limited image handling, so give it a try!

Find & Replace: This reveals a search bar at the bottom of the display. Type what you want to find and press Return. You can easily replace the found text with whatever you type in the replace field, or even replace all occurrences at once.

Find Next: If you've already searched for something, this jumps directly to the next occurrence.

Font: Not exactly what you think! This simply lets you toggle Bold, Italic, and <ins>Underline</ins> on your selected text.

Paragraph: Align your text (Left, Center, Right, Justify) or change the indentation (Indent, Unindent). Note: Left is sometimes called "ragged right", and Right is sometimes called "ragged left", just to confuse things.

Lowercase: Change the selected text to entirely lowercase.

Uppercase: Change the selected text to entirely uppercase.

Preferences: Lets you set your default program settings. This includes the editor font, autosave intervals, text-to-speech voices, and more. (More on preferences later in the tour!)

The Novel Menu
This menu handles the structure and organization of your novel's items.

Add Item: Adds a new item underneath your current one, allowing you to name it and assign tags.

Edit Item: Allows you to change the name and tags of an existing item.

Remove Item: Deletes an item. Don't worry, you can use Undo if you make a mistake. You are welcome!

Move Current Item Up: Moves the current item higher up in the document tree. If it reaches the top of its current parent group, it will "pop out" and move above its parent.

Move Current Item Down: Moves the item down the tree, or out of its parent group if it reaches the bottom.

Move Current Item Out: For when your nested lists get too deep, this simply promotes the item directly out of its current parent folder.

Open/Close Current Item: Expands the current item to see its children, or collapses it to hide them.

Open All: Expands every single item in your document tree. It can be nice to see the whole structure, or it might just be overwhelming.

Close All: If "Open All" was too much, this collapses everything. Take a deep breath and start opening things one at a time.

The View Menu
This menu controls how you look at your writing environment.

Distraction Free (F11): This brings up the distraction-free window. I develop TextSmith on a 4K monitor (actually, it's an old TV I got for free, which is great for vision issues), so my screen has a lot of empty space. Distraction-free mode dynamically adjusts to your monitor to give you a clean slate. You can drag the margins to make it look exactly like a page, and it completely blocks out the main TextSmith window until you hit ESC or F11 to exit.

Features in this mode: You can type, use basic text editing (cut, copy, paste, undo, redo, formatting), and check your word counts at the bottom of the left-hand bar. If you've enabled typing sounds in your preferences, you'll hear them here.

Read To Me: Starts reading the current item out loud using your operating system's selected text-to-speech voice. (If your OS doesn't have text-to-speech installed, it won't work—I can't program that part... yet!)

Word Counts: Displays your current word counts clearly in the left part of the bottom status bar so you can track your progress.

Spell Checking: TextSmith only checks your spelling when you explicitly ask it to. No distracting red squiggles while you write! When activated, it will highlight unrecognized words. (Currently loaded with an English dictionary, but more can be added if there's a demand!)

The Help Menu
About TextSmith: Displays a quick blurb about the program and shows my appreciation for everyone who helped me along the way.

Help: Opens this exact manual! Nice, huh?

Web Site: Takes you to my website so you can grab the most current version of TextSmith.

Scripting: Shows you the loaded and available scripts. These are tools you can use to automate particular tasks or add features I didn't include out of the box. (More on this in the scripting section!)

Part I: The Philosophy
The "One Object" Rule: Whether it's a Hero, a Castle, a Chapter, or a Scene—inside the engine, they are all just "Items."

The Tree is King: The visual hierarchy is the only structure that matters.

Why Tags Matter: Tags are the only way the engine knows that "Item A" is text to be printed and "Item B" is a character note to be ignored.

Part II: The Writer's Interface (The "How-To")
Creating Items: The basic building block.

Writing Content: The editor, formatting, and full-screen focus.

The "Scene" Tag: Telling the compiler "This is content."

The "Chapter" Tag: Telling the compiler "Start a new section."

Nesting: Putting Scenes inside Chapters (visual organization vs. tag logic).

Part III: The World Builder (Metadata)
Defining Your Schema: You aren't stuck with "Character" sheets. You define "NPC," "Villain," or "MacGuffin" using your own tags.

The "Bible" View: Storing world data (characters, locations) right next to the manuscript in the same tree.

Images: Using items as containers for maps and portraits.

Part IV: The Chameleon Compiler (Exporting)
Mapping Tags to Output: The core instruction set.

Default: Chapter = Heading + Break. Scene = Text.

Custom: Using shorthand (e.g., 'C' vs 'Chapter') and how the project remembers it.

The "Ignore" Rule: Anything untagged is invisible to the exporter.

Target-Specific Tags: PDF-Cover vs EBook-Cover.

Part V: Preferences & Defaults
Project Memory: How the system learns your specific tag style for this book.

Global Defaults: Setting the baseline for future projects.

Part VI: The Scripting Engine (The Aseembly Line)
The Parity Principle: "If you can do it with a mouse, you can do it with a script."

1. The Concept: The Stack of Plates
The Golden Rule: "Put data on the table, then tell the robot what to do with it.
"Visual Metaphor: Explain that the script works like a cafeteria stack. You put a "Scene" on the stack, then you put a "Command" on top to process it.
Reading Order: Explain why it reads Left-to-Right (Data $\to$ Action) rather than the reverse (Action $\leftarrow$ Data) they might remember from High School algebra.2.

2. Basic Verbs (The "Words")
Navigation: NextScene, PrevScene, GoToChapter.
Content: GetText, SetText, Insert.
Tags: GetTags, AddTag, RemoveTag.
Documentation Style: Don't use function signatures void addTag(string t). Use stack effects: ( Item Tag -- Item ) (Show what it takes, and what it leaves).

3. The "Nice to Have" Examples (Deconstructed)
Use your actual planned scripts as the tutorials.
The "Word Count" Tutorial:
    Explain the flow: CurrentItem $\to$ GetText $\to$ CountWords $\to$ ShowAlert.
The "Tag Injection" Tutorial:
    Explain the conditional: GetText $\to$ "Letter:" $\to$ Contains? $\to$ IF $\to$ "Correspondence" $\to$ AddTag $\to$ ENDIF.

4. Safety First
The Undo Sandwich: Explain that every script that modifies text must start with BeginUndo and end with EndUndo.
The "ReadOnly" Mode: Advise users to test scripts on a "Copy" of their item first.

5. The Dictionary (Reference)
Group commands by category (Movement, Editing, Logic, Math).
Crucial: Include a "Cheat Sheet" for stack manipulation (Dup, Swap, Drop). This is the hardest part for newbies ("Wait, why is my text gone? Oh, I forgot to Dup it before counting words.").

Automation:

Batch Tagging (e.g., "Tag all items in folder 'Drafts' as 'ToDo'").

Content Generation (e.g., "Create 10 items named 'Soldier 1' to 'Soldier 10'").

Formatting Cleanup (e.g., "Find all double spaces and replace with single").

The API Reference: The dictionary of commands that mirror the UI.


[Scripting ideas}

📄 Export & Formatting
- Scene Outline Export: generate a list of scene titles with summaries/tags.
- POV/Tag Filter Export: export only scenes tagged with a specified keyword.
- Timeline Export: collect scene dates/times from tags and output a chronological timeline.

🔍 Analysis & Consistency
- Word Count per Scene/Chapter: tally words and report totals for pacing analysis.
- Tag Frequency Report: count how often specific tags (e.g. POV, location) appear.
- Dialogue Ratio Check: calculate percentage of dialogue vs. narration in each scene.
- Character Mention Tracker: scan text for character names and produce a per‑scene presence map.

✍️ Text Manipulation
- Smart Find/Replace: replace words/phrases across all scenes with undo‑safe logging.
- Tag Injection: automatically add tags based on regex matches (e.g. detect “Letter:” and tag scene as correspondence).

🛠️ UI & Workflow Automation
- Menu Macro Runner: bind a script to a menu item that runs a sequence of editor commands.
- Batch Scene Reorder: reorder scenes based on tag criteria (e.g. timeline) instead of manual drag.
- Quick Outline Generator: insert a new document section with chapter/scene headings only.
- Delete Empty Scenes: scan tree and remove nodes with no text content.

🎁 Showcase / Fun
- Random Prompt Inserter: insert a writing prompt at the cursor from a supplied list.
- Mood Tagger: scan for sentiment words and tag scenes with “tone: dark/light/romantic.”
- Chapter Title Styler: auto‑apply formatting (caps, underline) to chapter headings.
